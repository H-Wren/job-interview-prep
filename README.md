# job-interview-prep

一个 [Claude Skill](https://docs.claude.com/en/docs/claude-code/skills)：把求职面试准备拆成四个可复用阶段，避免每投一家公司就从零分析一遍简历和JD。

## 这是什么

给 Claude（claude.ai / Claude Code / Claude Cowork）使用的结构化工作流程，覆盖：

- **Phase 0 · 候选人知识库** — 从简历、个人网站、GitHub、作品集等来源整理出「基本画像 / 核心竞争力 / STAR 案例库」，一次建好，后续每家公司都能复用，不用重复分析。这是一份持久化的 Markdown 文件，需要自己保存、下次手动带回对话里（不是 Claude 的自动记忆）。
- **Phase 1 · 公司与产品调研** — 搜集公司背景、近期动态、小红书职场口碑、面经；如果JD/公司涉及具体产品，会专门做产品深度调研（UI/UX 印象、优缺点、改进建议）。
- **Phase 2 · 简历 × JD 匹配分析** — 逐条对照JD要求，找出简历里不清晰/待优化的点，给具体改写建议，不只是泛泛的"写具体点"。
- **Phase 3 · 押题预测** — 分 HR 面 / 部门负责人面 / 产品测评 / 跨行业转型四类，每条押题都能追溯到具体的调研发现或简历细节，不是题库式套话。
- **Phase 4 · 参考答案预演** — 基于真实简历/知识库内容写参考回答；严格规定不编造信息，材料不够的地方会明确标注"这里需要你补充"。

## 安装

把 `job-interview-prep.skill` 安装到支持 Claude Skills 的环境（例如 Claude Code）。具体安装路径请以 Claude Code 当前版本的官方文档为准，不同版本可能不同。

仓库里同时保留了未打包的源文件（`job-interview-prep/SKILL.md`），方便直接阅读或修改 prompt 内容。

## 使用方式

直接告诉 Claude 要准备哪家公司的面试，附上 JD 和简历（第一次使用建议先建知识库，之后投递新公司时直接带上知识库文件即可）。

## 已知限制

- 公司/产品调研、面经搜集依赖联网搜索（Web Search）。未开启时 Claude 只能用已有知识回答，并会明确标注可能过时，不会编造调研结果。
- LinkedIn 内容无法直接抓取（登录墙限制），需要自己粘贴或导出后提供给 Claude。
- 候选人知识库是一份实际文件，不是自动记忆——每次新对话需要重新上传，或存放在 Claude Projects 的项目知识库里。
